Optimizing CSS

Next >>

In this lesson, I learnt about optimizing CSS for readability and easier maintenance.

Good, clean code is always a great idea. Here are some hot tips on optimizing your CSS:

Code better!

  1. Do the work right the first time. Don't rely on optimization tools to make you a good CSS3 coder.
  2. As one of my favourite sayings goes, "Crap in; Crap out".
  3. Use shallow selectors instead of long, redundant selectors.
  4. Utilize good code organization
  5. Use shorthand where possible
  6. Treat your CSS like it's your art.

Validate!

Always validate your code! Sometimes the smallest error can cause the biggest headache. There are awesome, free tools out there, such as this tool here.

Compress!

When you're developing, you want your CSS to be highly readable and easy to navigate, with comments and spacing. But in a production-ready site (i.e. a live client site, or app), you don't need all of that stuff! Create a production-only CSS file and add your highly-compressed CSS to that file when you launch! Use a free tool website like this one to compress your CSS.

If manually doing this process becomes a pain, use a tool that automatically optimizes & compresses your CSS, and creates a "production" file on your server. I highly suggest CodeKit.

Pre-load!

Did you know you could tell your browser to "pre-load" your CSS? This could improve performance and ensure your styles load quickly for users. Simply add the following to the <head> tag of your HTML when calling your CSS file:

<link rel="preload" href="app.css" as="style">

Lighten up your Frameworks!

Frameworks, like Bootstrap, are notoriously "heavy", meaning there are a lot of files, functions, assets and code that you simply do not need in your website. Perhaps you're just using a Framework for the Grid, the Base Styles, or a specific element, like Buttons!

If this is the case for you, when you download the Framework, either remove EVERYTHING you're not using, or download the specific files you need to make those resources quicker to load in the browser.

Learn a pre-processor language!

Take advantage of a CSS pre-processor, like Sass or LESS. They'll not only save you coding time and give you more styling flexibility, but you can export highly-optimized CSS for production-ready sites & applications!
